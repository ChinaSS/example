1.2.1.         Simple Mode

In this mode, we use one of the most simple way to deal with upload request, that means file parted and break point reverted are not supported, in other words, if you send a request for a single file, we only start one thread to process upload transaction. The URL is like this:

http://<yourhost>:<port>/<your servlet path>/?mode=simple

2.2.2.         Model design

We defined a base uploader, which has full upload business and it¡¯s an instance of WebUploader. It was registered on a lot of default method, and it supported to call the extend methods in EMO(Extend Methods Object), which created by its custom implementation.

Of cause, in order to provide convenience to the developer, we provided multiple file upload and single file upload instance. So you can use a simple way to create your own uploader.
